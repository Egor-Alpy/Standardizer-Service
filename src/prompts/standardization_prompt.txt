ЗАДАЧА: Стандартизировать характеристики товаров согласно предоставленным стандартам ОКПД2.

ИНСТРУКЦИИ:
1. Для каждого товара сопоставь его атрибуты со стандартными характеристиками
2. Используй ТОЛЬКО характеристики из предоставленного стандарта для данной группы ОКПД2
3. Сопоставляй атрибуты по смыслу, не требуется точное совпадение названий
4. Если атрибут не соответствует ни одной стандартной характеристике - НЕ включай его
5. Стандартизируй значения согласно списку values или приводи к единицам из units
6. Возвращай результат СТРОГО в формате JSON массива

ПРАВИЛА СТАНДАРТИЗАЦИИ:
- standard_name: точное название характеристики (ключ из стандарта)
- standard_value: значение из списка values ИЛИ число с единицей измерения из units
- characteristic_type: тот же ключ характеристики из стандарта
- Учитывай контекст товара при сопоставлении атрибутов
- Если точного значения нет в values - выбери наиболее подходящее
- Числовые значения приводи к единицам измерения из units

ПРИМЕРЫ СОПОСТАВЛЕНИЯ:
- "тип" может означать "Вид", "Тип чечевицы", "Категория" в зависимости от товара
- "количество слоев" → "Количество слоев" (если есть в стандарте для бумаги)
- "2-слойная" → standard_value: "2", если units содержит "слой"
- "белая" → найти подходящую характеристику типа "Цвет" и значение "белый"

ФОРМАТ ВЫВОДА:
[
  {
    "product_id": "ID товара из входных данных",
    "standardized_attributes": [
      {
        "standard_name": "точное название из ключа стандарта",
        "standard_value": "стандартизированное значение",
        "characteristic_type": "ключ характеристики"
      }
    ]
  }
]

ВАЖНО:
- Возвращай ТОЛЬКО JSON массив, без дополнительного текста
- Каждый товар должен быть обработан
- Пропускай атрибуты, которые не подходят под стандарт
